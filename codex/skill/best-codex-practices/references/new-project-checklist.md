# 新项目 Codex 配置清单（可直接复制）

## A. 最小初始化（必须）

1. 在仓库根目录创建或更新 `AGENTS.md`。
2. 在仓库根目录创建或更新 `.codex/config.toml`。
3. 在 `AGENTS.md` 写明可执行命令：
   - 安装/启动
   - lint / type-check / test / build
4. 在 `AGENTS.md` 写明 Definition of Done（至少含验证命令）。
5. 对“复杂任务”写明先计划（`/plan` 或 `PLANS.md`）再实施。

## B. 安全与权限（建议默认）

1. 从更严格权限开始（approval/sandbox 先紧后松）。
2. 禁止在源码硬编码密钥；统一环境变量读取。
3. 明确高风险操作确认机制（删库、覆盖、强推等）。

## C. 质量闭环（必须）

1. 改动后至少执行与改动最相关的检查。
2. 必要时补充或更新测试。
3. 交付时给出：
   - 改动摘要
   - 影响范围
   - 验证结果
   - 未验证项与风险

## D. MCP 与 Skill（按需）

1. 仅接入 1-2 个高价值 MCP（先验证收益再扩展）。
2. 对重复任务沉淀 skill（一个 skill 做一件事）。
3. skill 触发描述写清楚“做什么 + 何时触发”。

## E. 自动化（在稳定后）

1. 手动流程连续稳定后，再上 Automations。
2. 优先自动化：
   - 例行汇总
   - 质量巡检
   - 发布说明草拟

## F. 会话组织（长期收益）

1. 一条线程对应一个任务单元。
2. 分叉需求再 fork，不要把所有任务堆在一条长线程。
3. 主线程做决策，子线程做探索/测试/排查。
